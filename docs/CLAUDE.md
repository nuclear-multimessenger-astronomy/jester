## Building Documentation

```bash
# Build documentation locally
uv run sphinx-build docs docs/_build/html

# Open in browser
open docs/_build/html/index.html      # macOS
```

## Documentation Architecture

### Hybrid Approach

JESTER uses a **hybrid documentation system**:

1. **Auto-generated** - YAML configuration reference from Pydantic schemas
   - Source: `../jesterTOV/inference/config/schema.py`
   - Generator: `../jesterTOV/inference/config/generate_yaml_reference.py`
   - Output: `inference/yaml_reference.md`
   - **When to regenerate**: After modifying `schema.py`
   - **Command**: `uv run python -m jesterTOV.inference.config.generate_yaml_reference`

2. **Manual** - Narrative documentation, guides, examples
   - All other `.md` and `.rst` files
   - Must be updated manually when features change

## Formatting Standards

### Math in Docstrings

**Always use reStructuredText math formatting in docstrings:**

```python
def gamma_function(x: float) -> float:
    r"""Compute the gamma function.

    The gamma function is defined as :math:`\Gamma(x) = \int_0^\infty t^{x-1} e^{-t} dt`.

    For display math, use:

    .. math::
        \Gamma(x) = \int_0^\infty t^{x-1} e^{-t} dt

    Args:
        x: Input value where :math:`x > 0`
    """
```

**Key rules:**
- Use raw strings `r"""` for docstrings with LaTeX (avoids Python escape warnings)
- Inline math: `:math:\`expression\``
- Display math: `.. math::` directive
- See `../jesterTOV/eos/base.py` for reference examples

### MyST Markdown

**Cross-references** (use MyST syntax, NOT standard markdown links for internal refs):

```markdown
<!-- Referencing a section with an anchor -->
See {ref}`config-system` for details.

<!-- Creating an anchor (use explicit MyST style) -->
(config-system)=
## Configuration System
```

**Mermaid diagrams**:

````markdown
```{mermaid}
graph TD
    A[Config] --> B[Parser]
    B --> C[Sampler]
```
````

**NOT** `` ```mermaid`` (plain code fence) - must use MyST directive syntax.

## API Reference Structure

**Hierarchical organization** (inspired by sklearn):

1. **Top level** (`jesterTOV.eos.rst`) - Navigation only, lists submodules via `toctree`
2. **Mid level** (`jesterTOV.eos.metamodel.rst`) - Overview with `autosummary` listing classes
3. **Leaf level** (`jesterTOV.eos.metamodel.MetaModel_EOS_model.html`) - Individual class pages

### Module Overview Page Pattern

**Correct pattern** for module overview pages (e.g., `jesterTOV.eos.metamodel.rst`):

```rst
``jesterTOV.eos.metamodel`` module
===================================

.. currentmodule:: jesterTOV.eos.metamodel

Module description here (1-2 sentences).

Mathematical Background (optional)
-----------------------------------

Additional context, equations, etc.

Classes
-------

.. autosummary::
   :nosignatures:              # Clean table without signatures
   :toctree: _autosummary/     # Generate individual class pages

   ClassName1
   ClassName2
```

**Key principles:**
- **DO NOT** use `.. automodule::` directive - this expands all class details on the overview page
- **DO** use `.. currentmodule::` to set the module context
- **DO** use `:toctree: _autosummary/` to generate individual class pages
- **DO** keep overview text concise - just enough to understand what's in the module
- Individual class pages are auto-generated using the `_templates/class.rst` template

**Import structure:**
- Remove `__all__` re-exports from `__init__.py` files
- Use explicit imports: `from jesterTOV.eos.metamodel import MetaModel_EOS_model`
- This ensures Sphinx autodoc organizes classes by their actual module location

## Common Documentation Tasks

### Adding New Documentation

1. Create `.md` file in `docs/`
2. Add to appropriate `toctree` in `index.rst`:
   ```rst
   .. toctree::
      :maxdepth: 2
      :caption: Section Name

      new_document
   ```
3. Build locally to verify
4. Run strict build: `uv run sphinx-build -W --keep-going docs docs/_build/html`

### Fixing Sphinx Warnings

**Common warning types:**

1. **Undefined reference**: `WARNING: undefined label: 'section-name'`
   - Add explicit MyST anchor: `(section-name)=`
   - Use {ref} syntax: `{ref}\`section-name\``

2. **Citation not found**: `WARNING: Citation [citation1] is not referenced`
   - Add citation to `../jesterTOV/references.bib` or relevant docstring
   - Use correct citation format in docstrings

3. **Toctree reference not found**
   - Ensure file exists and is in correct location
   - Check filename matches exactly (case-sensitive)

4. **Ambiguous reference**
   - Add to `nitpick_ignore` in `conf.py` if spurious
   - Or make reference more specific

### Updating Configuration Documentation

**When you modify `../jesterTOV/inference/config/schema.py`:**

1. Make your changes to Pydantic models
2. Regenerate YAML reference:
   ```bash
   uv run python -m jesterTOV.inference.config.generate_yaml_reference
   ```
3. Review changes in `inference/yaml_reference.md`
4. Update manual docs if the change is user-facing:
   - `inference.md` - Conceptual explanations
   - `inference/quickstart.md` - If it affects quick start workflow
5. Commit both `schema.py` and `inference/yaml_reference.md` together

**Recommended**: See `inference_documentation_guide.md` for complete maintenance workflow.

## Common Pitfalls

### Section Landing Pages Pattern

**Problem**: Navigation breaks or sections disappear from sidebar when clicking through pages.

**Solution**: Copy `examples.rst` structure exactly:

```rst
Intro text.

Section Header
--------------

**Subsection Name**
   Description.

   :doc:`path/to/subsection`

.. toctree::
   :hidden:

   path/to/subsection
```

**Key rules**:
1. No top-level title with `===` (caption in index.rst provides this)
2. Section header with `---`
3. Toctree MUST be `:hidden:`
4. After structural changes, do clean rebuild: `rm -rf docs/_build && uv run sphinx-build docs docs/_build/html`

### Jupyter Notebooks with nbsphinx

**Requirement**: Notebooks must be located within the `docs/` directory structure for nbsphinx to process them.

**Problem**: Using `.. nbsphinx::` directive or external symlinks causes errors.

**Solution**:
1. Move notebooks to `docs/examples/` or appropriate subdirectory
2. Reference them directly in toctrees: `examples/eos_tov/notebook_name`
3. Ensure `pandoc` is installed: `brew install pandoc` (macOS) or `apt install pandoc` (Linux)

**Example**:
```rst
.. toctree::
   :hidden:

   examples/eos_tov/eos_tov
   examples/eos_tov/automatic_differentiation
```

## Known Issues

**LaTeX rendering**: Requires MathJax 3 (configured in `conf.py`). If math doesn't render, check browser console for MathJax errors.

**Mermaid diagrams**: Must use MyST directive syntax `{mermaid}`, not plain code fence. Requires `sphinxcontrib.mermaid` extension.
